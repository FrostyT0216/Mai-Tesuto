# 试题自测程序 — 改进需求文档

> 本文档基于忠实用户的长期使用反馈整理，按优先级分类，供产品迭代参考。

---

## 一、高优先级 — 核心答题体验优化

### 1.1 答题快捷键支持

**现状**：答题过程中所有操作均依赖鼠标点击（上一题、下一题、提交答案、选择选项），刷题效率受限于鼠标操作。

**改进方案**：
| 快捷键 | 功能 |
|--------|------|
| `←` | 上一题 |
| `→` | 下一题（已提交答案后） |
| `Enter` | 提交答案 |
| `1` / `2` / `3` / `4` / `5` | 直接选择对应选项 |
| `Space` | 下一题（等价于 `→`） |
| `Ctrl+Enter` | 多选题提交（选中多个选项后） |

注意这些快捷键都可以在设置的子页面中自定义绑定按键。

**验收标准**：
- 焦点不在输入框时，快捷键生效
- 添加快捷键提示卡片（在答题页底部或悬浮提示，可按 `?` 查看）
- 多选题中，按字母键切换选项的选中状态

---

### 1.2 练习进度持久化（断点续练）

**现状**：练习中途关闭页面或导航离开，所有答题进度丢失，无法恢复。

**改进方案**：
- 每次选项变化或提交答案时，自动将当前练习状态保存到 `localStorage`（或通过服务端 API）
- 保存内容：当前题库名称、已选题目列表、当前题号、每题答案状态（未答/已选/已提交/正确/错误）
- 下次进入练习设置页时，检测是否存在未完成的练习，弹出提示：
  > 「检测到上次未完成的练习（题库：XXX，进度 12/50），是否继续？」
- 提供 `继续上次练习` 和 `重新开始` 两个选项

**数据结构参考**：
```javascript
{
  bankName: "国防教育章节测验题库",
  quizMode: "random",
  questions: [...],      // 当前练习的题目列表
  currentIndex: 12,      // 当前所在题号
  answers: {             // 每题作答状态
    "0": { value: "A", submitted: true },
    "1": { value: "B", submitted: true },
    "2": undefined       // 未作答
  },
  savedAt: "2026-05-30T14:30:00"
}
```

---

### 1.3 跳题功能

**现状**：只能按顺序前后翻题，无法跳过某个题目稍后回来作答。

**改进方案**：
- 在答题操作区增加「跳过」按钮
- 跳过后的题目以特殊状态标记（如黄色/虚线边框）
- 完成最后一道题后，如有跳过的题目，自动跳回第一个未答的题目
- 或者：允许用户选择「提交时是否包含跳过的题目（按错误计算）」

---

### 1.4 题号导航面板

**现状**：仅通过 `第 X / N 题` 文本和进度条展示进度，无法快速跳转。

**改进方案**：
- 在答题页顶部（进度条下方）增加题号导航条
- 使用小方块/圆形数字网格展示所有题号
- 颜色编码：
  - 🟢 绿色：已答对
  - 🔴 红色：已答错
  - ⚪ 灰色：未作答
  - 🟡 黄色：已跳过
- 点击任意题号直接跳转
- 移动端横向滚动显示

## 二、中优先级 — 统计与分析增强

### 2.1 统计概览仪表盘

**现状**：统计页分散在正确率、错题本、练习历史三个 tab，缺乏整体视图。

**改进方案**：
- 新增「概览」tab（或作为统计页默认展示）
- 展示内容：
  - 核心指标卡片：总练习次数、总正确率、连续打卡天数
  - 薄弱章节排行（按正确率从低到高排列的章节列表）
  - 近期趋势迷你图（近 48小数/7 天正确率走势，可用简单柱状图或折线图）
  - 本周/本月刷题数量统计

---

### 2.2 「未做过」练习模式

**现状**：仅有顺序、随机、易错题三种模式。

**改进方案**：
- 新增「未做过」模式（或「刷新题」模式）
- 筛选当前统计数据中 `total_attempts === 0` 的题目
- 与章节筛选联动，可选择只刷新题
- 在练习设置的「练习模式」卡片中增加第三个选项

---

### 2.3 答题时显示题目历史统计

**现状**：答题时不显示该题的历史作答情况。

**改进方案**：
- 在题目卡片右上角或题目类型徽章旁边，显示该题历史数据
- 格式示例：`📊 做过 5 次 · 正确率 60%`
- 颜色根据正确率高低变化
- 可在设置中开关此显示

---

### 2.4 错题本练习模式

**现状**：错题本只能查看，无法单独导出。

**改进方案**：
- 在答题页面增加「错题本练习」按钮
- 这个模式中只含有错题本中的题目

---

## 三、低优先级 — 体验优化与扩展

### 3.1 搜题功能

**现状**：题库量大时，在统计或章节页面无法快速定位某道题。

**改进方案**：
- 在章节页面和统计页面顶部增加搜索框
- 支持按题干关键词实时过滤
- 搜索结果高亮匹配文字
- 搜索结果保留所属章节信息

---

### 3.2 题目收藏 / 星标

**现状**：只有错题会被标记，经典好题无法单独收藏。

**改进方案**：
- 答题时增加「收藏」按钮（星标图标）
- 收藏的题目在统计页有独立 tab 或筛选项
- 收藏数据纳入统计数据，随导出一起保存
- 在题目详情弹窗中也可切换收藏状态

---

### 3.3 计时模式

**现状**：无时间限制，无法模拟考试场景。

**改进方案**：
- 练习设置页增加「限时模式」开关和时长输入（分钟）
- 答题页显示倒计时器
- 时间到后：
  - 自动提交当前答案并结束练习
  - 弹出提示「时间到！练习已自动提交」
- 限时练习在历史记录中特殊标记（如显示 ⏱️ 图标）

---

### 3.4 纯浏览模式

**现状**：想快速翻阅题目和答案复习时，必须进入答题→提交→看解析的流程。

**改进方案**：
- 在练习设置中增加「浏览模式」选项
- 进入后直接展示题目、所有选项、正确答案（高亮）、解析
- 不记录答题统计
- 支持左右翻页、章节跳转
- 适合考前快速过题

---

### 3.6 移动端适配优化

**现状**：基本可用但触控体验不够好。

**改进方案**：
- 增大移动端选项按钮的触控区域（至少 44px 高度）
- 底部操作按钮改为固定吸底（`position: sticky` 优化）
- 筛选项适配小屏（Chip 组件换行更紧凑）
- 题号导航面板在移动端改为横向滚动

## 四、细节改进

### 4.1 多选题提交按钮状态

**现状**：多选题选中选项后，提交按钮无任何状态变化，用户可能忘记选没选够就点击。

**改进方案**：
- 多选题选中至少一个选项后，提交按钮从禁用/普通状态变为高亮（如增加轻微脉冲动画或边框发光）
- 未选中任何选项时，提交按钮显示为略灰色状态
